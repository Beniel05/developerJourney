# Markdown Documentation References

## Always Check the Documentation

When creating notes, README files, project documentation, or GitHub profiles, refer to Markdown documentation if you forget the syntax.

### 1. Markdown Guide (Recommended)
- Website: https://www.markdownguide.org/basic-syntax/
- Best for:
  - Headings
  - Lists
  - Links
  - Images
  - Tables
  - Code blocks
  - Blockquotes
- Beginner-friendly and easy to navigate.

### 2. GitHub Flavored Markdown (GFM)
- Website: https://github.github.com/gfm/
- Best for:
  - GitHub README files
  - Tables
  - Task lists
  - Strikethrough
  - Advanced GitHub Markdown features
- Official specification used by GitHub.

### 3. GitHub Docs - Markdown
- Website: https://docs.github.com/en/get-started/writing-on-github
- Best for:
  - README files
  - GitHub profile README
  - Issues and Pull Requests
  - GitHub-specific formatting
- Official GitHub documentation.

## Recommended Approach

1. Use Markdown Guide for learning syntax.
2. Use GitHub Docs when writing README files.
3. Refer to GitHub Flavored Markdown for advanced formatting.
4. Practice by creating:
   - README.md
   - Project documentation
   - Personal notes
   - Portfolio documentation

> Knowing Markdown is a superpower for developers because almost every repository, documentation site, and technical note uses it.
