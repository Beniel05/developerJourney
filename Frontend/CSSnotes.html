Types of CSS

CSS can be applied in three ways: Inline, Internal, and External. Each has different use cases depending on project size and maintainability.

1. Inline CSS

<h1 style="color: blue;">Hello World</h1>
  • Scope: One element
  • Use: Quick fixes
  • Problem: Not scalable

2. Internal CSS

<style>
  p { color: red; }
</style>
  • Scope: Single page
  • Use: Small projects
  • Problem: Not reusable

3. External CSS

<link rel="stylesheet" href="styles.css">
  • Scope: Multiple pages
  • Use: Real-world apps
  • Best Practice: Always preferred

CSS Selectors

CSS selectors are used to target HTML elements for styling. They act as a bridge between HTML and CSS.

1. Core Selectors

Universal Selector (*)

* { margin: 0; padding: 0; }

Targets all elements. Used for reset.

Element Selector

p { color: blue; }

Targets all elements of a type.

Class Selector (.class)

.card { border: 1px solid #ccc; }

Reusable and recommended for styling.

ID Selector (#id)

#header { background: black; }

Used for unique elements. Avoid overuse.

Combinators (Relationships)

Descendant

div p { color: red; }

Targets all p inside div.

Child

div > p { color: green; }

Only direct children.

Adjacent Sibling

h1 + p { color: orange; }

First p after h1.

General Sibling

h1 ~ p { color: purple; }

All p after h1.

Advanced Selectors

Attribute Selector

a[target="_blank"] { color: red; }

Pseudo-class

button:hover { background: blue; }

Pseudo-element

p::first-letter { font-size: 30px; }

Key Principles

Specificity Priority:

Inline > ID > Class > Element

Grouping:

h1, h2, p { color: blue; }

Summary Comparison

Feature Inline Internal External
Scope Single Page Multiple
Priority Highest Medium Lowest
Maintainability Hard Moderate Easy

CSS Units (Absolute vs Relative)

In CSS, units are categorized into absolute (fixed size) and relative (scale based on another value).

1. Pixels (px) & Default Browser Sizes

The px is an absolute unit. In most modern browsers, the default root font size is 16px.

Element Default Size (em) Equivalent Pixels (px)
h1 2em 32px
h2 1.5em 24px
h3 1.17em 18.72px
h4 1em 16px (Bold)
h5 0.83em 13.28px
h6 0.67em 10.72px
p 1em 16px

2. Relative Units: rem, em, and %

rem (Root em)

  • Relative to the root (<html>) font size
  • Use: Typography & spacing
  • Benefit: Scales with user accessibility settings

em

  • Relative to parent element
  • Use: Component-based sizing
  • Risk: Compounds (can grow unpredictably)

% (Percentage)

  • Relative to parent property
  • Use: Layout widths & responsive design
  • Benefit: Fluid layouts

rem → root based

em → parent based (can compound)

% → parent size

3. Best Practices

  • Typography: Use rem for consistency & accessibility
  • Layout: Use %, vw, vh
  • Small Details: Use px (borders, shadows)
  • Component Spacing: Use em

Example

html {
  font-size: 16px;
}

h1 {
  font-size: 2rem; /* 32px */
}

.container {
  width: 80%;
}

.button {
  font-size: 1rem;
  padding: 0.5em 1em;
}

4. em vs rem

Feature em rem
Relative To Parent Root
Scaling Compounds Stable
Predictability Hard Easy
Best Use Components Typography 
.parent { font-size: 20px; }
.child { font-size: 1.5em; } /* 30px */
.grandchild { font-size: 1.5em; } /* 45px 😬 */
      
html { font-size: 16px; }
.child { font-size: 1.5rem; } /* always 24px */

5. Viewport Units

  • vw → 1% of width
  • vh → 1% of height
  • vmin → smaller side
  • vmax → larger side
.hero {
  width: 100vw;
  height: 100vh;
}

Mobile Fix:

height: 100dvh;

CSS Colors (Color Theory)

Colors in CSS define the visual appearance of elements. They can be represented in multiple formats like RGB, RGBA, and Hexadecimal.

0. How Colors Actually Work (Additive Color Theory)

Screens use RGB (Additive Color Model). Colors are created by mixing light.

  • Red + Green = Yellow
  • Red + Blue = Magenta
  • Blue + Green = Cyan
  • Red + Green + Blue = White
  • No color (0,0,0) = Black

👉 This is why rgb(255,255,0) gives yellow.

1. RGB (Red, Green, Blue)

RGB defines colors using 3 values from 0 to 255.

color: rgb(255, 0, 0); /* Red */
  • 255 → full intensity
  • 0 → no color
  • Example: rgb(255,255,0) → Yellow

2. RGBA (With Transparency)

RGBA adds alpha (opacity) from 0 to 1.

background: rgba(0, 0, 255, 0.5);
  • 1 → fully visible
  • 0 → fully transparent
  • Used in overlays, modals, glass UI

3. Hexadecimal (#RRGGBB)

Hex uses base-16 values from 00 to FF.

color: #ff0000; /* Red */
  • #000000 → Black
  • #ffffff → White
  • #ff0000 → Red
  • Short form: #fff

4. RGB vs HEX (Mental Model)

RGB is easier to understand logically, while HEX is shorter and widely used. 

rgb(255, 0, 0)  → Red  
#ff0000         → Same Red

5. Comparison

Format Example Use Case
RGB rgb(255,0,0) Learning & clarity
RGBA rgba(255,0,0,0.5) Transparency
HEX #ff0000 Production

8. Real World Usage Tips

  • Use HEX → most common in projects
  • Use RGBA → overlays & shadows
  • Use RGB → debugging & learning
  • Avoid random colors → use a palette

9. How Many Colors Are Possible? (16.7 Million 🎨)

In RGB, each color channel (Red, Green, Blue) can have values from 0 to 255.

  • Red → 256 possible values
  • Green → 256 possible values
  • Blue → 256 possible values

Total combinations:

256 × 256 × 256 = 16,777,216 colors

👉 That’s 16.7 million possible colors in standard RGB!

Example Breakdown

  • rgb(0, 0, 0) → Black (no light)
  • rgb(255, 255, 255) → White (full light)
  • rgb(255, 0, 0) → Red
  • rgb(0, 255, 0) → Green
  • rgb(0, 0, 255) → Blue

Why 256?

Each color channel uses 8 bits:

2⁸ = 256 values (0–255)

So:

3 channels × 8 bits = 24-bit color

👉 That’s why standard displays are called 24-bit color systems.

Fun Insight!

Even though 16.7 million sounds huge, modern displays can show even more using techniques like HDR and 10-bit color.
